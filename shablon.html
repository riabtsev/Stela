§ 03 / Process

Three steps. Your code never leaves your machine.

01 Submit

Encrypted manifest

You send a three-part manifest — system prompt, orchestration code, model name. Encrypted with the enclave’s public key inside your browser, before transit.

  • system_prompt.txt
  • orchestration.{py,ts}
  • model_id (vendor-agnostic)
→ ciphertext only beyond this line
02 Validate in TEE

Sealed environment

Validation runs inside a Trusted Execution Environment. Three test groups exercise the agent. No human — including us — ever sees your code.

  • Reliability · task completion under load
  • Safety · adversarial prompt injection
  • Alignment · stays inside stated scope
→ only the result exits the enclave
03 Attest on-chain

Permanent record

Report is pinned to IPFS. Its hash is written into a smart contract via ERC-8004. You receive a verifiable link and a badge — neither expires.

  • IPFS CID + Arweave mirror
  • ERC-8004 Validation Registry
  • Public badge for docs / repo / RFP
✓ anyone can verify, independently
